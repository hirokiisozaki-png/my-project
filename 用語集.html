AGV
Automated Guided Vehicle｜無人搬送車
搬送機器 基本用語
決められたルートに沿って自動で走るフォークリフトや台車のような搬送車両
詳細を見る
【特徴】 ・人が運転しなくても自動走行 ・磁気テープ・QRコード・誘導線などの"決められたルート"を走行 ・障害物があると停止する安全機能あり ・一定のルートを繰り返す作業に向いている 【用途】 ・倉庫での棚入れ・棚出し ・工場での部品運搬 ・作業者への部材供給 ・完成品の搬送 【AMRとの違い】 AGV：決められたルートのみ走行（ガイドが必要） AMR：周囲を認識し自律的にルートを選択（よりスマート）
🤖
AMR
Autonomous Mobile Robot｜自律走行搬送ロボット
搬送機器 基本用語
周囲の環境を自分で認識し、目的地まで最適なルートを自律的に走行できるロボット
詳細を見る
【特徴】 ・地図（SLAM）を作成し、自分の位置を認識 ・障害物を避けてルートを再計算して走行 ・ルート敷設（磁気テープ・ラインなど）が不要 ・人や他のロボットと共存しやすい ・柔軟にレイアウト変更に対応できる 【用途】 ・倉庫内での商品搬送 ・ピッキング支援 ・工場内の部品供給 ・病院・オフィスでの物品配送 【AGVとの比較】 項目　　 AGV　　　　　　 AMR 走行　　 決められたルートのみ　自律的にルート選択 変更への対応 ルート変更に弱い　 レイアウト変更に柔軟 技術　　 比較的シンプル　　センサー・AIで高度 コスト　　安い　　　　　 高め
🏗️
AGF
Automated Guided Forklift｜自動運転フォークリフト
搬送機器
自動で走って荷物の上げ下ろしもできるフォークリフト（AGVのフォークリフト版）
詳細を見る
【特徴】 ・無人で走行（AGVのフォークリフト版） ・パレットの持ち上げ・棚入れ・棚出しが自動 ・センサーで位置を認識して安全に運転 ・既存のフォークリフト作業を自動化できる 【AGV/AMRとの違い】 AGV：荷物を運ぶ台車型 AMR：自律判断して動くロボット AGF：フォークリフト機能（リフトアップ）が付いたタイプ

WMS
Warehouse Management System｜倉庫管理システム
物流システム 基本用語
倉庫の在庫がどこにあって、何が入ってきて、何が出ていったかを正確・効率的に管理するツール
詳細を見る
【主な機能】 ・在庫の場所・数量をリアルタイムで把握 ・入荷・出荷作業の指示や管理 ・ピッキング（商品の取り出し）の最適化 ・誤発送や在庫ミスの削減 倉庫作業を自動化・効率化して、コスト削減や作業スピード向上に役立てるシステム。 WES・WCS・RCSのシステム階層の最上位に位置する。
⚡
WES
Warehouse Execution System｜倉庫実行システム
物流システム
ロボットなどに「今どの仕事をどう進めるか」指示を出す現場の司令塔
詳細を見る
【WMSとの違い】 WMS：在庫や入出荷情報を「管理」する WES：作業者やマシン（AGV・ロボット等）に「今どの仕事をどう進めるか」を指示する 【WESが得意なこと】 ・作業の優先順位づけ ・AGV・AMR・ソーターなど機器との連携 ・作業負荷のバランス調整 ・ボトルネックのリアルタイム解消 ・ピッキングや仕分けの最適ルート計算 人とロボットの動きを統合し、倉庫全体の流れを最適化するのがWESの役割。
🔧
WCS
Warehouse Control System｜倉庫制御システム
物流システム
倉庫の機械（コンベア・ソーター等）をリアルタイムに動かす「装置のコントローラー」
詳細を見る
【役割】 ・コンベアを動かす／停止させる ・ソーターで荷物を振り分ける ・自動倉庫（AS/RS）に棚入れ・棚出しを指示 ・AGV・AMRの動作ルートを制御 ・機器の稼働状況を監視 【WMS・WES・WCSの位置づけ】 レイヤー 役割 例えるなら WMS　　 在庫・入出荷情報の管理 "計画担当" WES　　 作業の最適化・指示 "現場の司令塔" WCS　　 機械設備の直接制御 "機械の操作係"
🎮
RCS
Robot Control System｜ロボット制御システム
物流システム
倉庫内のロボット（AMR・AGV等）の動作を統合的に管理・制御するシステム（ロボット版のWCS）
詳細を見る
【役割】 ・ロボットの位置管理 ・ルートの最適化 ・交通制御（渋滞・衝突回避） ・タスクの割り当て ・稼働状況の監視 「ロボット版のWCS（倉庫制御システム）」と理解するとわかりやすい。
🗂️
物流システム 階層構造
WMS → WES → WCS → RCS → AGV の流れ
物流システム 図解
WMS→WES→WCS→RCS→AGV が正しいシステムの指示の流れ
WMS ── 倉庫管理システム（在庫・計画） ↓ 作業指示 WES ── 倉庫実行システム（最適化・司令塔） ↓ 設備制御指示 WCS ── 倉庫制御システム（コンベア・ソーター） ↓ 搬送タスク RCS ── ロボット制御システム（群制御） ↓ 走行命令 AGV/AMR/AGF ── 自動搬送機器（実行）

VDA 5050
AGV/AMR 標準通信プロトコル
通信・技術 国際標準
メーカーに依存せず、AGV/AMRを標準化された方法でつなぐための通信ルール
詳細を見る
ドイツ自動車工業会（VDA）とVDMAが共同制定した、AGV/AMRと上位システム間の標準通信プロトコル。 【目的】 従来はメーカーごとに通信仕様がバラバラだったため、異なるメーカーのAGVを同じRCSで管理できなかった。VDA 5050はこの問題を解決するための共通ルール。 【メリット】 ・異なるメーカーのAGV/AMRを同じRCSで統合管理 ・API仕様の個別開発が不要 ・導入コスト・開発コストの大幅削減 ・欧州では多くの現場が導入済み 【技術仕様】 ・JSONベースのメッセージング ・MQTTで送受信 ・通信対象：RCS ↔ AGV/AMR本体 【主にやり取りされる情報】 マップ情報、タスク（搬送指示）、動作状態、エラー情報、位置情報

SLAM
Simultaneous Localization and Mapping｜自己位置推定と地図作成
通信・技術
ロボットが「今自分がどこにいるか」を把握しながら「周囲の地図」を作る技術。ロボットの目と脳みそ。
詳細を見る
【わかりやすい例え：初めて入る暗い部屋】 懐中電灯を持って暗い部屋に入ったとき—— 1. 地図を作る（Mapping）：周囲を照らして「右に壁がある」「前に机がある」と間取りを把握 2. 自分の位置を知る（Localization）：作った間取りから「自分は今ドアの近くにいるな」と特定 この2つを同時にリアルタイムで行うのがSLAM。 【営業上のメリット】 ・床の工事（磁気テープ）が不要→導入ハードルが大幅低下 ・障害物を避けて自律走行→人が立っていても迂回 【2つの方式の比較】 種類 LiDAR SLAM（レーザー） Visual SLAM（カメラ） 暗闇 ◎ 全く問題なし × 苦手 ガラス × 透過・乱反射で苦手 〇 認識できる 精度 高い（数センチ単位） やや低め コスト 高価 比較的安価 弱点 景色・色は分からない 光の変化・同一景色に弱い

詳細を見る
【役割】 ・センサー入力を受け取る ・プログラム通りに判断する ・モーターやバルブなどへ出力する 【例え】 制御盤：家 PLC：家の中にある頭脳（司令塔）

API連携
Application Programming Interface
通信・技術
別々のシステムやアプリ同士が、自動で情報のやり取りをする仕組み。「システム同士の共通の窓口」。
詳細を見る
【PLC連携 vs API連携の比較】 項目 PLC連携 API連携 通信 接点・フィールドバス Ethernet/IP 情報量 少ない 多い 状態把握 ON/OFF中心 詳細ステータス 拡張性 低め 高い IT連携 苦手 得意 【使い分けの原則】 安全・リアルタイム・現場完結 → PLC制御 情報量・柔軟性・拡張性 → API連携 【実務的な最適解：ハイブリッド構成】 ・安全・動作の最終制御 → PLC ・状態管理・判断・指示 → API 「人の安全や確実な動作はPLCで担保し、ロボットの判断や設備間の連携はAPIで行う。役割を分けることで安全性と柔軟性を両立。」

イーサネット
Ethernet｜有線ネットワーク規格
通信・技術
有線でネットワーク機器をつなぐときの世界共通のルール。Wi-Fiより遅延が少なく安定。
詳細を見る
【ポイント3つ】 ① ケーブルでつなぐ有線通信 　Wi-Fiが「無線」なのに対し、イーサネットは「有線」。LANケーブルを差し込む方式。 ② データ送受信のルール 　データを小さな塊（パケット）に分けて送り、相手が正しく受け取れるよう手順が決まっている。 ③ 速くて安定している 　Wi-Fiより遅延が少なく安定しているため、会社のオフィスや工場の現場でよく使われる。 【よく見るもの】 ・LANケーブル　　　→ イーサネットで通信するためのケーブル ・LANポート（RJ-45）→ パソコンやルーターにある差し込み口 ・スイッチングハブ　→ 複数機器をつなぐ中継装置

コミッショニングエンジニア（CE）
現場導入の設計・施工担当エンジニア
業務
AGV/AMR導入現場における「設計・段取り・施工」の実務担当者
詳細を見る
【デスクワーク系の役割（設計・段取り・司令塔）】 ・現地調査（現調）：現場の床の段差・通路幅・Wi-Fi電波・壁や柱の配置などを詳細測定 ・レイアウト・コース設計：ロボットの走行ルート（マップ）を専用ソフトで設計 ・関係各所との調整：SIerやエレベーター・シャッター等の設備メーカーとシステム打ち合わせ 【現場作業系の役割（施工・職人）】 ・機体のセットアップ：組み立て、センサー・カメラの感度調整 ・ティーチング（マッピング）：ロボットを手動で動かして地図を覚え込ませ、数センチ単位で走行設定 ・現場テスト・トラブルシューティング：実荷物で走らせ、エラー原因を突き止めてプログラム修正 ・操作説明：現場作業員への操作方法・緊急停止時の対応レクチャー
